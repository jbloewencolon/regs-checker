# Regs Checker — Phased Remediation & Optimization Plan

**Companion to:** [`docs/production_readiness_review.md`](production_readiness_review.md) (2026-07-01 audit)
**Goal:** Close the four launch blockers, then clean and optimize the repository into a maintainable, fast, production-grade system.
**Structure:** 8 phases. Phases 0–3 are strictly sequential gates (each has exit criteria that block the next). Phases 4–7 can overlap once Phase 2 lands. Task IDs are stable — reference them in commits (`fix(P1-2): ...`).

Effort key: **S** ≤ half day · **M** 1–2 days · **L** 3–5 days.
Every phase ends with a **Gate** — the checks that must pass before the phase is called done.
