How confidence scores work

Each extraction receives a composite confidence score (0–100%) built from up to six weighted components. The score determines the tier (A ≥ 85%, B ≥ 70%, C ≥ 50%, D < 50%).

ORRICK GATE: Extractions without Orrick validation data are automatically Tier D regardless of other scores.

ComponentWeightWhat it measures
■ Schema Validity10%Did the LLM output conform to the expected Pydantic schema? 0% means structurally broken.
■ Evidence Grounding20%Fraction of extracted fields with a verified evidence span pointing back to source text.
■ Completeness10%Proportion of optional fields the model filled in. Missing fields lower this score.
■ Source Quality5%Parse quality from the ingestion pipeline. Low scores indicate OCR errors or truncated content.
■ Orrick Alignment30%Token overlap with Orrick reference data. Required — without this, the extraction is auto-Tier D.
■ Cross-Validation25%Accuracy from the cross-validation agent that independently checks extracted facts against the source.

When Cross-Validation is unavailable, its weight is redistributed proportionally among the active components. Orrick Alignment is never redistributed — it is required.
